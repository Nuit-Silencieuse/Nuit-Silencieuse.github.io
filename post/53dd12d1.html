这篇文章总结了Docker在Linux平台的使用方式。

Docker简介

从虚拟化层看容器，轻量级、高性能是核心价值

  • 容器是在Linux内核实现的轻量级资源隔离机制

  • 虚拟机是操作系统级别的资源隔离，容器本质上是进程级的资源隔离

启动MySQL容器

执行命令如下：

docker pull mysql:8.0.36：拉取镜像，最好要注明镜像的版本号，否则是latest版本，不一定有很好的兼容性或稳定性。

一次性的、不使用数据卷的MySQL容器启动（可省略）

docker run -it -p 3306:3306 --name mysql -e MYSQL_ROOT_PASSWORD=123456 mysql:8.0.36：启动容器，以交互式运行。

docker cp mysql:/var/log mysqldata/log：复制容器内log文件夹的内容至宿主机。

docker cp mysql:/var/lib/mysql mysqldata/data：复制容器内mysql数据至宿主机。

docker cp mysql:/etc/mysql/conf.d mysqldata/conf：复制容器内mysql配置至宿主机。

docker cp mysql:/docker-entrypoint-initdb.d mysqldata/init：复制容器内mysql初始化脚本至宿主机。

以上复制到宿主机的内容作为新的mysql容器的初始化配置，以后对于mysql的配置更改、日志分析都应该在这些磁盘上的文件夹内进行。

docker stop mysql ，docker remove mysql 停止并移除原先的容器。

持久化的、使用数据卷的MySQL容器启动

docker network create hm-net：创建通用网络。

重新启动mysql容器：cd ~

详细解释：

​ /var/lib/mysql 存储数据信息；/etc/mysql/conf.d文件夹内的mysql配置文件.cnf；/docker-entrypoint-initdb.d文件夹内存放初始化脚本.sql。

​ -v 宿主机位置:容器内位置，其中“宿主机位置”必须以./开头，因此必须注意命令执行时的当前目录。

​ --restart=always 表示自启。

其他命令：

docker exec -it mysql mysql -uroot -p：进入MySQL命令行。第一个mysql 是容器名称。

常用命令

docker exec -it CONTAINER_ID bash：进入容器内部文件系统的终端。CONTAINER_ID 是容器名称。

网络

docker network create hm-net：创建通用网络。

docker network ls：列出docker网络。

日志

docker logs -f CONTAINER_ID：查看容器日志。

容器运行

docker inspect CONTAINER_ID：查看容器的详细配置信息。

docker inspect -f '{{range .NetworkSettings.Networks}}{{.IPAddress}}{{end}}' container_name_or_id：查看容器的ip地址。

docker rm -f CONTAINER_ID：删除容器，-f是允许强制删除正在运行的容器。
